# Modelfile 레퍼런스

Modelfile은 Ollama로 사용자화된 모델을 만들고 공유하기 위한 청사진(설계도)입니다.
이 문서는 Modelfile의 형식과 각 지시어(instruction)의 사용법을 정리합니다.

## 형식

Modelfile의 기본 형식은 다음과 같습니다.

```
# comment
INSTRUCTION arguments
```

사용할 수 있는 지시어는 다음과 같습니다.

| 지시어 | 설명 |
|---|---|
| `FROM` (필수) | 사용할 베이스 모델을 정의합니다. |
| `PARAMETER` | Ollama가 모델을 실행하는 방식을 제어하는 파라미터를 설정합니다. |
| `TEMPLATE` | 모델에 전달할 전체 프롬프트 템플릿입니다. |
| `SYSTEM` | 템플릿에 설정될 시스템 메시지를 지정합니다. |
| `ADAPTER` | 모델에 적용할 (Q)LoRA 어댑터를 정의합니다. |
| `LICENSE` | 법적 라이선스를 지정합니다. |
| `MESSAGE` | 메시지 기록을 지정합니다. |
| `REQUIRES` | 모델이 요구하는 Ollama의 최소 버전을 지정합니다. |

## 예시

### 기본 Modelfile

마리오 캐릭터를 흉내 내는 모델을 만드는 예시입니다.

```
FROM llama3.2
# sets the temperature to 1 [higher is more creative, lower is more coherent]
PARAMETER temperature 1
# sets the context window size to 4096, this controls how many tokens the LLM can use as context to generate the next token
PARAMETER num_ctx 4096

# sets a custom system message to specify the behavior of the chat assistant
SYSTEM You are Mario from super mario bros, acting as an assistant.
```

사용 방법:

1. 위 내용을 파일로 저장합니다(예: `Modelfile`).
2. `ollama create choose-a-model-name -f <파일 위치, 예: ./Modelfile>`
3. `ollama run choose-a-model-name`
4. 모델을 사용하기 시작합니다.

기존 모델의 Modelfile을 확인하려면 `ollama show --modelfile` 명령을 사용합니다.

```shell
ollama show --modelfile llama3.2
```

출력 예시:

```
# Modelfile generated by "ollama show"
# To build a new Modelfile based on this one, replace the FROM line with:
# FROM llama3.2:latest
FROM /Users/pdevine/.ollama/models/blobs/sha256-00e1317cbf74d901080d7100f57580ba8dd8de57203072dc6f668324ba545f29
TEMPLATE """{{ if .System }}<|start_header_id|>system<|end_header_id|>

{{ .System }}<|eot_id|>{{ end }}{{ if .Prompt }}<|start_header_id|>user<|end_header_id|>

{{ .Prompt }}<|eot_id|>{{ end }}<|start_header_id|>assistant<|end_header_id|>

{{ .Response }}<|eot_id|>"""
PARAMETER stop "<|start_header_id|>"
PARAMETER stop "<|end_header_id|>"
PARAMETER stop "<|eot_id|>"
PARAMETER stop "<|reserved_special_token"
```

## 지시어

### FROM (필수)

`FROM` 지시어는 모델을 만들 때 사용할 베이스 모델을 정의합니다.

```
FROM <model name>:<tag>
```

#### 기존 모델로부터 빌드

```
FROM llama3.2
```

사용 가능한 베이스 모델 목록은 [GitHub 모델 라이브러리](https://github.com/ollama/ollama#model-library)에서,
추가 모델은 [Ollama 라이브러리](https://ollama.com/library)에서 찾을 수 있습니다.

#### Safetensors 모델로부터 빌드

```
FROM <model directory>
```

모델 디렉터리에는 지원되는 아키텍처의 Safetensors 가중치가 들어 있어야 합니다.

현재 지원되는 모델 아키텍처:

- Llama (Llama 2, Llama 3, Llama 3.1, Llama 3.2 포함)
- Mistral (Mistral 1, Mistral 2, Mixtral 포함)
- Gemma (Gemma 1, Gemma 2 포함)
- Phi3

#### GGUF 파일로부터 빌드

```
FROM ./ollama-model.gguf
```

GGUF 파일 위치는 절대 경로 또는 Modelfile 기준 상대 경로로 지정합니다.

### PARAMETER

`PARAMETER` 지시어는 모델 실행 시 설정할 파라미터를 정의합니다.

```
PARAMETER <parameter> <parametervalue>
```

#### 유효한 파라미터와 값

| 파라미터 | 설명 | 타입 | 사용 예시 |
|---|---|---|---|
| `num_ctx` | 다음 토큰을 생성하는 데 사용할 컨텍스트 윈도우 크기를 설정합니다. (기본값: 2048) | int | `num_ctx 4096` |
| `repeat_last_n` | 반복을 막기 위해 모델이 얼마나 이전까지 거슬러 볼지 설정합니다. (기본값: 64, 0 = 비활성화, -1 = `num_ctx`) | int | `repeat_last_n 64` |
| `repeat_penalty` | 반복에 대한 페널티 강도를 설정합니다. 값이 높을수록(예: 1.5) 반복을 강하게 억제하고, 낮을수록(예: 0.9) 관대합니다. (기본값: 1.1) | float | `repeat_penalty 1.1` |
| `temperature` | 모델의 온도입니다. 온도를 높이면 모델이 더 창의적으로 답합니다. (기본값: 0.8) | float | `temperature 0.7` |
| `seed` | 생성에 사용할 난수 시드를 설정합니다. 특정 값으로 고정하면 같은 프롬프트에 대해 같은 텍스트를 생성합니다. (기본값: 0) | int | `seed 42` |
| `stop` | 정지 시퀀스를 설정합니다. 이 패턴을 만나면 LLM이 텍스트 생성을 멈추고 반환합니다. `stop` 파라미터를 여러 번 지정하면 여러 정지 패턴을 설정할 수 있습니다. | string | `stop "AI assistant:"` |
| `num_predict` | 텍스트 생성 시 예측할 최대 토큰 수입니다. (기본값: -1, 무제한 생성) | int | `num_predict 42` |
| `draft_num_predict` | 드래프트 모델이 있을 때 한 단계당 예측할 추측성(speculative) 드래프트 토큰의 최대 수입니다. 별도 드래프트 모델은 기본 4이며, 임베디드 MTP 텐서는 이 값을 설정해야 합니다. 0으로 설정하면 추측성 드래프팅이 비활성화됩니다. | int | `draft_num_predict 4` |
| `top_k` | 무의미한 출력을 줄입니다. 값이 높으면(예: 100) 더 다양한 답을, 낮으면(예: 10) 더 보수적인 답을 줍니다. (기본값: 40) | int | `top_k 40` |
| `top_p` | top-k와 함께 동작합니다. 값이 높으면(예: 0.95) 더 다양한 텍스트를, 낮으면(예: 0.5) 더 집중되고 보수적인 텍스트를 생성합니다. (기본값: 0.9) | float | `top_p 0.9` |
| `min_p` | top-p의 대안으로, 품질과 다양성의 균형을 맞추는 것을 목표로 합니다. `p`는 가장 확률 높은 토큰 대비 어떤 토큰이 고려되기 위한 최소 확률을 뜻합니다. 예를 들어 `p`=0.05이고 가장 확률 높은 토큰의 확률이 0.9라면, 0.045 미만의 로짓은 걸러집니다. (기본값: 0.0) | float | `min_p 0.05` |

### TEMPLATE

`TEMPLATE`은 모델에 전달할 전체 프롬프트 템플릿입니다. 선택적으로 시스템 메시지, 사용자 메시지,
모델 응답을 포함할 수 있습니다. 문법은 모델마다 다를 수 있습니다. 템플릿은 Go의
[템플릿 문법](https://pkg.go.dev/text/template)을 사용합니다.

#### 템플릿 변수

| 변수 | 설명 |
|---|---|
| `{{ .System }}` | 사용자 정의 동작을 지정하는 시스템 메시지 |
| `{{ .Prompt }}` | 사용자 프롬프트 메시지 |
| `{{ .Response }}` | 모델의 응답. 응답을 생성할 때 이 변수 뒤의 텍스트는 생략됩니다. |

```
TEMPLATE """{{ if .System }}<|im_start|>system
{{ .System }}<|im_end|>
{{ end }}{{ if .Prompt }}<|im_start|>user
{{ .Prompt }}<|im_end|>
{{ end }}<|im_start|>assistant
"""
```

### SYSTEM

`SYSTEM` 지시어는 템플릿에서 사용할 시스템 메시지를 지정합니다(해당하는 경우).

```
SYSTEM """<system message>"""
```

### ADAPTER

`ADAPTER` 지시어는 베이스 모델에 적용할 파인튜닝된 LoRA 어댑터를 지정합니다. 어댑터 값은 절대
경로 또는 Modelfile 기준 상대 경로여야 합니다. 베이스 모델은 `FROM` 지시어로 지정해야 합니다.
어댑터가 튜닝된 베이스 모델과 지정한 베이스 모델이 다르면 동작이 불안정해집니다.

#### Safetensor 어댑터

```
ADAPTER <path to safetensor adapter>
```

현재 지원되는 Safetensor 어댑터:

- Llama (Llama 2, Llama 3, Llama 3.1 포함)
- Mistral (Mistral 1, Mistral 2, Mixtral 포함)
- Gemma (Gemma 1, Gemma 2 포함)

#### GGUF 어댑터

```
ADAPTER ./ollama-lora.gguf
```

### LICENSE

`LICENSE` 지시어로 이 Modelfile과 함께 사용되는 모델이 공유·배포되는 법적 라이선스를 지정할 수 있습니다.

```
LICENSE """
<license text>
"""
```

### MESSAGE

`MESSAGE` 지시어로 모델이 응답할 때 참고할 메시지 기록을 지정할 수 있습니다. `MESSAGE` 명령을
여러 번 사용해 대화를 구성하면, 모델이 비슷한 방식으로 답하도록 유도할 수 있습니다.

```
MESSAGE <role> <message>
```

#### 유효한 역할(role)

| 역할 | 설명 |
|---|---|
| `system` | 모델에 시스템 메시지를 제공하는 또 다른 방법 |
| `user` | 사용자가 물어볼 법한 메시지 예시 |
| `assistant` | 모델이 응답해야 하는 방식의 메시지 예시 |

#### 대화 예시

```
MESSAGE user Is Toronto in Canada?
MESSAGE assistant yes
MESSAGE user Is Sacramento in Canada?
MESSAGE assistant no
MESSAGE user Is Ontario in Canada?
MESSAGE assistant yes
```

### REQUIRES

`REQUIRES` 지시어로 모델이 요구하는 Ollama의 최소 버전을 지정할 수 있습니다.

```
REQUIRES <version>
```

버전은 유효한 Ollama 버전이어야 합니다(예: 0.14.0).

## 참고 사항

- **Modelfile은 대소문자를 구분하지 않습니다.** 예시에서 지시어를 대문자로 쓴 것은 인자와 구분하기
  쉽게 하기 위함입니다.
- 지시어는 어떤 순서로든 둘 수 있습니다. 예시에서 `FROM`을 맨 앞에 둔 것은 읽기 쉽게 하기 위함입니다.
