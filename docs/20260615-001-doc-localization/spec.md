# doc-localization

외부 기술 문서 사이트를 설정(config) 기반으로 수집해 한국어로 번역·미러링하는 도구. 올라마(docs.ollama.com)를
첫 타깃 겸 레퍼런스 구현으로 삼고, 이후 다른 문서 사이트도 같은 틀로 추가할 수 있게 한다.

## 승인 전 확인
- 번역 출력을 번역문 단독으로 둘지(현재 가정), 원문 병기로 바꿀지. 관련 본문: §1, §4
- 대상 언어를 한국어 단일로 고정할지(현재 가정), 다국어 확장을 완료 조건에 포함할지. 관련 본문: §4

## 1. 범위
- 외부 문서 사이트의 페이지 목록을 확보하고, 각 페이지의 마크다운 원문을 수집해 로컬에 보존한다.
- 수집한 원문을 사람이 읽기 좋은 한국어 기술 문서로 재구성·번역해 산출한다. 원문이 OpenAPI 스펙 등
  기계친화 포맷이면 그 내용을 표·설명 등 가독성 있는 구조로 재구성한다. 파일 단위의 디렉터리 구조는
  원본을 따르되, 문서 내부 구성은 가독성을 위해 원문과 달라질 수 있다.
- 사이트마다 달라지는 값(베이스 URL, 페이지 목록 확보 방식, 포함·제외 패턴, 출력 경로, 용어집 경로)을
  사이트별 설정 파일로 분리해, 수집기·번역 절차 본체를 사이트 간에 재사용한다.
- 페이지 목록 확보 방식(source 타입)은 `llms.txt` / `sitemap` / `crawl` 세 종류를 하나의 인터페이스로 다룬다.
  이번 범위에서 실제 동작하는 것은 `llms.txt` 타입이며, 나머지는 인터페이스 상에 정의만 두고 미구현으로 남긴다.
- 올라마를 `llms.txt` 타입의 레퍼런스 사이트로 포함한다.
- 수집은 결정적(deterministic) 작업으로 보고 Go 도구가 담당한다. 번역은 Claude가 파일을 읽어 수행한다.

## 2. 목표
- 영어로만 제공되는 기술 문서를, 사람이 읽기 좋게 가공한 한국어 문서로 로컬에서 확보한다.
- 새 문서 사이트를 추가할 때 코드 변경 없이(`llms.txt` 타입 한정) 설정 파일과 용어집만 추가하면 되도록 한다.
- 문서가 갱신되어도 전체를 다시 처리하지 않고 변경분만 갱신할 수 있게 한다.
- 용어 번역을 사이트별 용어집으로 고정해, 페이지·재실행 간 번역어 일관성을 유지한다.

## 3. 제약
- 수집 도구는 Go로 작성하며, 별도 런타임 설치 없이 단일 실행 파일로 동작한다(win32 우선).
- 번역의 실행 주체는 Claude(커맨드가 Claude 본인에게 파일을 읽혀 번역)이며, 외부 번역 API를 호출하지 않는다.
- 수집·번역은 증분(증분 갱신) 방식을 지원한다: 재실행 시 이미 처리된·변경되지 않은 항목은 다시 처리하지 않는다.
- 원본 마크다운은 로컬에 보존한다(재현·증분 비교·diff 용도).
- 외부 사이트에 대한 수집은 과도한 요청을 피한다(요청 간격·재시도 등 rate limit 고려).
- 사이트별 설정과 용어집은 도구·커맨드 본체와 분리된 파일로 둔다.

## 4. 제외 범위
- HTML 크롤링 후 본문 추출·마크다운 변환 파이프라인은 이번 범위에서 다루지 않는다
  (대상이 마크다운 원문을 제공하는 사이트이며, 그렇지 않은 사이트를 위한 `crawl` 타입은 인터페이스만 둔다).
- `sitemap` / `crawl` source 타입의 실제 동작 구현은 이번 범위 밖이다.
- 한국어 외 언어로의 번역(다국어 출력)은 이번 범위 밖이다.
- 번역문에 원문을 함께 싣는 병기(bilingual) 출력은 이번 범위 밖이다(번역문 단독으로 산출).
- 번역 품질 자동 평가·회귀 검사는 이번 범위 밖이다.
- 산출된 번역 문서를 외부에 게시·배포하는 단계는 이번 범위 밖이다.

## 5. 완료 조건
1. 사이트별 폴더 하나가 한 사이트를 나타낸다(폴더명이 사이트 식별자). 폴더 안의 설정 파일에 베이스 URL,
   source 타입, 포함·제외 패턴이 지정되고, 같은 폴더 안에 용어집·원문·번역문·매니페스트가 함께 저장된다
   (출력·용어집·매니페스트 위치는 폴더 규약으로 고정된다). 올라마용 사이트 폴더가 존재한다.
2. 수집을 실행하면 `llms.txt` 타입 사이트(올라마)의 페이지 목록에 해당하는 마크다운 원문이 그 사이트 폴더
   안의 원문 보관 위치에 저장되고, 저장된 파일이 원본의 경로 구조를 반영한다.
3. 수집을 다시 실행하면 변경되지 않은 원문은 다시 내려받지 않고, 새로 추가되거나 변경된 원문만 갱신된다
   (재실행 결과로 갱신된 항목 수가 관찰된다).
4. 번역을 실행하면 수집된 원문이 사람이 읽기 좋은 한국어 문서로 재구성·번역되어, 그 사이트 폴더 안의
   번역문 출력 위치에 파일 단위로 원본 디렉터리 구조를 보존해 저장된다. 원문이 OpenAPI 스펙 등 기계친화
   포맷이면 그 내용이 표·설명 등 가독성 있는 구조로 가공되며, 원문을 그대로 옮긴 기계친화 덩어리로 남지 않는다.
5. 번역을 다시 실행하면 이미 번역된·원문이 바뀌지 않은 페이지는 다시 번역하지 않고, 미번역분·변경분만 처리된다.
6. 번역 시 해당 사이트의 용어집에 정의된 용어는 그 정의된 번역어로 일관되게 반영된다.
7. `sitemap` / `crawl` source 타입이 설정 값으로 식별은 되며, 선택 시 "미구현"이 명확히 드러난다
   (조용히 실패하거나 `llms.txt`로 잘못 처리되지 않는다).
8. 새 `llms.txt` 타입 사이트를 추가할 때, 수집·번역 본체 코드를 수정하지 않고 사이트 폴더(설정 파일·용어집
   포함) 추가만으로 해당 사이트의 수집·번역이 동작한다.
