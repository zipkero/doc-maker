  - 접근: 베이스 URL·포함/제외 패턴을 입력받아 대상 페이지 목록과 각 페이지 원문 취득 경로를 산출하는

  - 검증 조건:
    - 결과: 타입 `sitemap`/`crawl` 선택 시 미구현임이 드러나는 오류로 중단하고, 타입 `llms.txt`는
      구현체로 분기한다.

      구현체 반환).
  - 참조: SPEC §5.7 / ANALYSIS §3, §5 D7

  - 검증 조건:

    - 확인: 패턴 적용 전후 목록을 비교하는 단위 테스트가 통과한다.

- [x] task-006: 원문 취득과 rate limit
  - 목적: 대상 페이지의 마크다운 원문을 외부 사이트에 과도한 요청을 피하며(요청 간격·재시도) 내려받고,
    취득 실패 시 이미 보관된 직전 원문을 보존한다.
  - 접근: 페이지별로 원문을 HTTP로 취득하되 요청 간격과 재시도를 두고, 재시도 후에도 실패한 페이지는
    갱신 실패로 표시하고 기존 원문을 덮어쓰지 않는다.
  - 검증 조건:
    - 결과: 정상 응답은 원문을 반환하고, 일시 오류는 재시도하며, 최종 실패 시 기존 원문이 보존된다.
    - 확인: 모킹된 HTTP(정상·일시 오류·최종 실패)에 대한 취득 동작 테스트가 통과한다.
  - 참조: SPEC §5.2 / ANALYSIS §2

- [x] task-007: 수집 증분 판정과 사이트 폴더 원문 저장
  - 목적: 수집을 실행하면 원문이 사이트 폴더 안 `raw/` 아래에 원본 경로 구조를 반영해 저장되고,
    다시 실행하면 변경되지 않은 원문은 다시 내려받지 않고 신규·변경분만 갱신되며 갱신 항목 수가 보고된다.
  - 접근: 취득한 원문 해시를 매니페스트 기록과 대조해 신규·변경만 골라 사이트 폴더 `raw/`에 원본 경로
    구조로 저장하고 매니페스트를 갱신하며, 변경 없는 항목은 재취득을 건너뛰고 갱신 건수를 출력한다.
  - 검증 조건:
    - 결과: 1회차 실행은 대상 원문을 모두 저장하고 경로 구조가 원본을 반영하며, 원문 무변경 상태의 2회차
      실행은 갱신 0건을 보고하고, 일부 원문 변경 시 변경분만 갱신 건수로 보고된다.
    - 확인: 1회차→2회차(무변경)→일부 변경 시나리오의 통합 테스트가 통과한다(갱신 건수·저장 경로 검증).
  - 참조: SPEC §5.2, §5.3 / ANALYSIS §2, §5 D5, D8

- [x] task-008: 수집 실행 진입점(사이트 지정 인자)

    - 확인: 올라마 식별자로 수집을 실행해 원문 저장과 갱신 건수 출력을 확인한다.

    - 결과: 미번역 페이지와 원문 해시가 바뀐 페이지는 대상에 포함되고, 이미 번역된 무변경 페이지는 제외된다.
    - 확인: 미번역·무변경·변경 세 상태 페이지가 섞인 입력에 대해 대상 선별 결과를 확인한다.

- [x] task-010: 용어집 적용 한국어 재구성·번역과 구조 보존 저장
  - 목적: 대상 원문을 사람이 읽기 좋은 한국어 문서로 재구성·번역하되(OpenAPI 등 기계친화 포맷은 표·설명
    으로 가공, 코드 예시는 유지), 용어집 정의 용어는 일관 반영하고, 파일 단위로 원본 디렉터리 구조를
    보존한 사이트 폴더 안 `output/`에 번역문 단독으로 저장한다.
  - 접근: 용어집의 원어→한국어 매핑을 적용해 내용을 한국어 문서로 재구성·번역하고, 사이트 폴더 `output/`에
    원본 경로 구조로 저장한 뒤 매니페스트에 "번역한 원문 해시"를 기록한다.
  - 검증 조건:
    - 결과: 출력 위치에 사람이 읽기 좋게 재구성된 한국어 문서가 저장되고(기계친화 원문은 원문 덩어리로
      남지 않고 표·설명으로 가공됨), 용어집 정의 용어가 반영되며, 매니페스트에 번역 원문 해시가 기록된다.
    - 확인: 기계친화 포맷(OpenAPI yaml)과 산문이 섞인 표본 원문을 처리해 출력물이 읽기 좋은 한국어
      문서로 가공됐는지, 용어 반영·출력 경로·구조·매니페스트 기록을 확인한다.
  - 참조: SPEC §5.4, §5.6 / ANALYSIS §2, §5 D1, D2, D5, D9, D11

  - 참조: SPEC §5.5 / ANALYSIS §2, §5 D8

- [x] task-013: 번역 실행 진입점(대상 출력·완료 기록 CLI)
  - 목적: 사이트 식별자로 번역 대상 목록을 출력하고, 번역이 끝난 페이지의 매니페스트 번역 해시를 기록하는

    (2) 번역문이 출력 위치에 존재하는 페이지의 번역 해시를 현재 원문 해시로 기록하는 명령을 둔다. 기존
